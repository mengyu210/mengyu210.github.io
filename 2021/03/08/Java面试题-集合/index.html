发布于 2021-03-08   更新于 2021-03-23 集合 面试题

集合相关

1.ArrayList和Vector的区别

相同点

1.都实现了List接口
2.底层实现都为数组,允许对数据进行快速随机访问，查询数据比较快，插入，删除数据比较慢
3.都维护查询的顺序，可以通过索引对元素进行随机访问
4.都允许空值
不同点
1.ArrayList线程不安全，Vector线程安全,用synchronized实现线程安全，即某一时刻只能有一个线程访问Vector,避免多线程同时写而引起的不一致性,但是实现同步需要花费时间，因此访问Vector比访问ArrayList慢

ArrayList的扩容率为50%，Vector的扩容率为100%

ArrayList:ArrayList int newCapacity = oldCapacity + (oldCapacity >> 1);

Vector: int newCapacity = oldCapacity + ((capacityIncrement > 0) ? capacityIncrement : oldCapacity);

2.说说ArrayList,Vector,LinkedList的存储性能和特性

1.ArrayList和Vector底层都是数组实现，LinkedList底层是双向链表实现的
2.ArrayList和Vector查找修改快，LinkedList插入删除快
3.Vector线程安全，ArrayList、LinkedList线程不安全

3.快速失败(fail-fast)和安全失败(fail-safe)的区别是什么？

区别

1.java.util包下面的所有集合类都是快速失败，java.util.concurrent的所有类都是安全失败

2.快速失败的迭代器会抛出ConcurrentModificationException，而安全失败的迭代器永远不会抛出这个异常，因安全失败的迭代器是基于底层集合做拷贝，它不受源集合修改的影响。

4.HashMap的数据结构

1.JDK1.7及之前采用数组+链表
2.JDK1.8采用数组+链表+红黑树
3.当链表长度大于等于8时会转成红黑树

5.HashMap的工作原理是什么？

HashMap基于hashing原理，我们通过put()和get()方法储存和获取对象。当我们将键值对传递给put()方法时，它调用键对象的hashCode()方法来计算hashcode，让后找到bucket位置来储存值对象。当获取对象时，通过键对象的equals()方法找到正确的键值对，然后返回值对象。HashMap使用链表来解决碰撞问题，当发生碰撞了，对象将会储存在链表的下一个节点中。 HashMap在每个链表节点中储存键值对对象。

6.HashMap时候进行扩容呢？

HashMap的默认的增长因子为0.75，默认初始化大小为16，当HashMap达到总容量的0.75倍，即达到HashMap的临界值大小时，就会发生扩容。扩容是耗性能的操作，需要重建Hash表，重新计算每个元素在数组中的位置。
初始化容量要设置成expectedSize/0.75 + 1的话，可以有效的减少冲突也可以减小误差。

10.Java集合类框架的基本接口有哪些？

基本接口有，Collection、List、Set、Map
可以分为两大接口，Collection 和Map，

一个元素集合一个键值对集合。

1.Collection 单列集合的根接口
2.List元素有序，可重复
3.ArrayList 可变长度的数组，适合查询，不适合增删
4.LinkedList 底层是双向循环链表，适合增删，不适合查询
5.Set 元素无序，不可重复
6.HashSet 根据对象的Hash值确定元素在集合中的位置
7.TreeSet 以二叉树的方式存储元素，实现对集合中的元素进行排序
8.Map 双列集合的根接口，用于存储键值映射关系的元素。
9.HashMap用于存储键值映射关系，不能出现重复的健key
10.TreeMap 用于存储键值映射关系，不能出现重复的键key，所有键按照二叉树的方式排序

11.HashSet和TreeSet有什么区别？

HashSet

1.不能保证元素的排列顺序，能插入null值，但只能插入一个
2.HashSet是基于HashMap实现的
3.HashSet的底层是哈希表实现的
TreeSet
1.保证元素的排列顺序，不能存空值
2.TreeSet是基于TreeMap实现的，不过Set用的只是Map的key
3.TreeSet的底层采用的是二叉树(红黑树)的数据结构

12.HashSet的底层实现是什么？

1.HashSet是基于HashMap实现的，底层是有哈希表支持的
2.特点是存储快，线程不安全

3.无序集合，不保证元素的存储循序，没有索引，不能存储重复元素，可以存Null值

运行原理

HashSet添加元素的时候会先调用HashCode得到元素的哈希值，然后通过经过异或移位等运算算出该元素在哈希表的的实际位置。
1.如果算出的元素存储的位置目前没有任何元素存储，那么该元素直接存储在该位置上
2.如果算出的元素存储的位置已经存在有其他的元素了，那么还会调用该元素的epuals方法与该位置的元素进行比较一次，如果返回为true视为重复元素不添加，如果返回为false，则允许添加。

14.为什么集合类没有实现Cloneable和Serializable接口？

克隆（cloning）或者序列化（serialization）的语义和含义是跟具体的实现相关的。因此应该由集合类的具体实现类来决定如何被克隆或者序列化

15.什么是迭代器

Iterator接口提供了很多对集合元素进行迭代的方法，每个集合类都包含了可以返回实例的迭代器方法，迭代器可以在迭代过程中删除集合中的元素，但是不能用集合中的remove方法，而是使用迭代器中的remove方法

16.Iterator和ListIterator的区别是什么？

1.ListIterator实现了Iterator接口
2.ListIterator添加了增加元素，替换元素，获取前一个和后一个元素的索引的方法
3.Iterator只支持前序遍历，ListIterator支持前序遍历和后续遍历
4.Iterator可以对set和List迭代，ListIterator只能对List迭代

17.数组Array和列表ArrayList有什么区别，什么时候应该使用Array而不是ArrayList?

1.Array长度固定，ArrayList动态扩容
2.Array支持基本类型和对象类型，ArrayList只支持对象类型
3.ArrayList提供很多对集合操作的方法

18.Java集合类框架的最佳实践有哪些

1.元素固定的情况下，选用Array类型而不是ArrayList类型
2.有些集合类型允许指定初始容量，如果能估计出存储的元素的数目来指定初始化容量，来避免重新计算hash值或者扩容。
3.为了类型安全、可读性和健壮性，使用泛型来避免运行时的ClassCastException。
4.底层的集合实际上是空的情况，返回长度为0的集合和数组，而不是null。
5.编程的时候接口优于实现

19.Comparable和Comparator 接口是干什么的？列出它们的区别

Comparable和Comparator的接口是用来实现比较排序的

区别

1.Comparable是一个内部排序接口也可以称为自然排序，即实现该接口的类支持排序，可以通过Collections.sort(),或者Arrays.sort()来进行排序
2.Comparator是一个外部排序接口也可以称为定制排序接口，即通过Collecctions.sort()进行排序时，需要传入比较器的实现。
3.当有两个比较器同时存在的时候，采用定制排序Comparator进行排序
3.基本数据类型String,Interger等默认实现了Comparable排序接口

20.Collection和Collections的区别

Collection 是List,Set等集合接口的父接口，它提供了对集合对象进行基本操作的通用接口方法

Collections 是util包下面的关于集合的工具类，它提供对集合排序，搜索，以及线程安全的各种操作

21.ArrayList 与 LinkedList 的不区别？

ArrayList底层的数据结构是数组实现，支持数据随机访问
LinkedList底层的数据结构是双向的链表实现，不支持数组的随机访问
使用下标访问一个元素，ArrayList的时间复杂度是O(1)，而LinkedList是O(n)

22.用哪两种方式来实现集合的排序？

使用Collections.sort()方法实现集合的排序。
使用TreeSet和TreeMap来存储数据

23.Java 中怎么打印数组？

Arrays.toString() 基本类型的数组
Arrays.deepToString() Object类型的数组

24.Java 中的 LinkedList 是单向链表还是双向链表？

双向链表

25.Java 中的 TreeMap 是采用什么树实现的？(答案)

红黑树

26.Hashtable 与 HashMap 有什么不同之处？

1.Hashtable不允许空值,HashMap只允许一个空值
2.Hashtable的底层Dictionary，HashMap的底层是Map
3.Hashtable是同步的，比较慢，HashMap没有同步策略，所以快

27.Java 中的 HashSet，内部是如何工作的？

HashSet的底部使用HashMap实现,由哈希表实现
由于 Map 需要 key 和 value，所以所有 key 的都有一个默认 value。类似于 HashMap，HashSet 不允许重复的key，只允许有一个 null key，意思就是 HashSet 中只允许存储一个 null 对象。

29.我们能自己写一个容器类，然后使用 for-each 循环码？

可以，你可以写一个自己的容器类。如果你想使用 Java 中增强的循环来遍历，你只需要实现 Iterable 接口。如果你实现 Collection 接口，默认就具有该属性。

ArrayList 和 HashMap 的默认大小是多数？

ArrayList默认大小为10
HashMap默认大小为16

WeakHashMap 是怎么工作的？

WeakHashMap 的工作与正常的 HashMap 类似，但是使用弱引用作为 key，意思就是当 key 对象没有任何引用时，key/value 将会被回收。

Java 中 LinkedHashMap 和 PriorityQueue 的区别是什么？

PriorityQueue 保证最高或者最低优先级的的元素总是在队列头部，但是LinkedHashMap 维持的顺序是元素插入的顺序。当遍历一个 PriorityQueue时，没有任何顺序保证，但是 LinkedHashMap 课保证遍历顺序是元素插入的顺序。

poll() 方法和 remove() 方法的区别？

oll() 和 remove() 都是从队列中取出一个元素，但是 poll() 在获取元素失败的时候会返回空，但是 remove() 失败的时候会抛出异常。
